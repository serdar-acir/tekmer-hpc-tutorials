## Reporting a Problem

To ensure fast and accurate support, include the following details:

### Required Information
- Your **username**
- Date and time of the issue
- Login node or compute node used (if applicable)

### Job Details
- Job ID (if submitted via Slurm)
- Submission script (`sbatch` / `srun` script)
- Command or application executed

### Error Information
- Full error message (copy-paste, not screenshot)
- Output and error files (`.out`, `.err`)

### File and Path Information
- Full paths to:
  - Input files
  - Scripts
  - Output directories

---

## Good vs Bad Example

**Bad:**
> My job is not working.

**Good:**
> Username: user01  
> Job ID: 123456  
> Command: sbatch run.sh  
> Error: "MPI_Init failed"  
> Output file: /home/user01/job.out  

---

## Response Expectations

- Initial response time: typically within **1 business day**
- Complex issues may require further investigation
- You may be asked for additional logs or test runs

---

## Best Practices

To get faster support:
- Always include reproducible examples
- Test on small datasets before large jobs
- Keep your job scripts organized and documented
- Use standard modules and supported software versions
